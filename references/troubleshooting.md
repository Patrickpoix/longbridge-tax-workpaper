# 故障排查

- 安装失败：使用 Python 3.11—3.13，在技能根目录执行 `python -m pip install .`。
- 测试导入失败：不要手工设置 `PYTHONPATH`；先执行 `python -m pip install -e ".[dev]" -c constraints.txt`。
- 加密 PDF 无法打开：确认 `LONGBRIDGE_PDF_PASSWORD`，不要通过命令行参数传密码。
- `unknown_template`：先安装 `python -m pip install ".[ocr]"` 后重试。若 OCR 后仍失败，保留 PDF，不继续生成结构化税务结果；为新布局增加能力锚点、表头别名和匿名回归 fixture，不要按年份放宽整个模板。
- PDF 字体或文字层异常：程序会检测空文本、连续方框/替换字符并自动尝试 OCR。OCR 页面图像只写入临时目录并自动删除；文本层与 OCR 冲突时以复核状态呈现，不静默猜测。
- 未指定年度时报错：只有完整 1—12 月年度才会被自动选择；可补齐月结单或显式指定 `--tax-year` 生成不完整年度底稿。
- 人民币金额为空：检查是否同时提供了 USD/CNY 与 HKD/CNY 年末汇率；未知汇率不会写成 0。
- 多账户：使用 `--account-id` 分开运行。
- 期初成本不完整：补充纳税年度之前、从开户或首次取得相关标的开始的月结单。
- 缓存未命中：只有 `pdf_extracts/<YYYY-MM>/manifest.json` 的文件名和 SHA-256 与当前 PDF 完全一致才会使用缓存。
- 重复运行：程序排除输出目录并按 PDF SHA-256 去重；若仍发现重复，检查是否把旧结果复制到输入目录外的其他未排除位置。

## 版式漂移与未来年份

- 历史观察：同一账户的月结单版式会随时间演进（例如早期的内联概览、后来的带表头账户概览、再到带含借权益价值的版本），表头字段会新增或减少，字体也可能变化。
- 容错方式：识别基于归一化后的能力锚点与表头别名（繁简转换、康熙部首、常见同义表头），而非硬编码年份或固定坐标；因此 2026/2027 及以后的轻度措辞变化通常仍能被识别。
- 新增别名：若未来出现全新措辞，在 `template_registry.py` 的别名元组（如 `STATEMENT_TITLE_ALIASES`、`TRADE_HEADER_ALIASES`）中补充同义词，并添加匹配的匿名回归 fixture；不要按年份整体放宽模板。
- 低置信度处理：仅靠 OCR 回退识别或版式特征得分过低的月份会被 `月结单版式识别置信度` 检查升级为 `REVIEW_REQUIRED`，提示人工复核表头与数值映射，不会静默接受。
